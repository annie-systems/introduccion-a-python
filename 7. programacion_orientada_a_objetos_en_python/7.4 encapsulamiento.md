## 7.4. Encapsulamiento
---
El **encapsulamiento** es una t√©cnica que consiste en ocultar los atributos y m√©todos internos de una clase al exterior, de modo que solo el propio objeto pueda acceder a ellos¬π. Esto permite proteger la integridad de los datos y evitar que sean modificados o accedidos de forma indebida por otras clases o funciones¬≤.

En Python, el encapsulamiento se puede simular precediendo los atributos y m√©todos con dos guiones bajos __, lo que indica que son "privados" y no se pueden acceder desde fuera de la clase¬π. Por ejemplo:

```python
class Persona:
    def __init__(self, nombre, edad):
        self.__nombre = nombre # Atributo privado
        self.__edad = edad # Atributo privado
    
    def getNombre(self): # M√©todo p√∫blico para acceder al atributo privado
        return self.__nombre
    
    def getEdad(self): # M√©todo p√∫blico para acceder al atributo privado
        return self.__edad
    
    def __saludar(self): # M√©todo privado
        print(f"Hola, soy {self.__nombre} y tengo {self.__edad} a√±os.")
    
    def presentarse(self): # M√©todo p√∫blico que llama al m√©todo privado
        self.__saludar()
```

En este ejemplo, la clase Persona tiene dos atributos privados: __nombre y __edad, y dos m√©todos privados: __init__ y __saludar. Estos elementos solo se pueden usar dentro de la propia clase, y no se pueden acceder desde fuera. Para poder obtener o modificar los atributos privados, se deben crear m√©todos p√∫blicos que hagan de interfaz, como getNombre, getEdad o presentarse. Estos m√©todos se pueden llamar desde cualquier parte del programa y pueden usar los atributos o m√©todos privados de la clase. Por ejemplo:

```python
p = Persona("Ana", 25) # Se crea un objeto de la clase Persona
print(p.getNombre()) # Se llama al m√©todo p√∫blico getNombre
print(p.getEdad()) # Se llama al m√©todo p√∫blico getEdad
p.presentarse() # Se llama al m√©todo p√∫blico presentarse, que a su vez llama al m√©todo privado __saludar

# Salida:
# Ana
# 25
# Hola, soy Ana y tengo 25 a√±os.
```

Sin embargo, si intentamos acceder directamente a los atributos o m√©todos privados desde fuera de la clase, obtendremos un error:

```python
print(p.__nombre) # Error: el atributo __nombre es privado
p.__saludar() # Error: el m√©todo __saludar es privado

# Salida:
# AttributeError: 'Persona' object has no attribute '__nombre'
# AttributeError: 'Persona' object has no attribute '__saludar'
```

Como curiosidad, en realidad s√≠ que podr√≠amos acceder a los elementos privados de la clase haciendo un poco de trampa. Python cambia el nombre de los atributos y m√©todos privados a√±adiendo el nombre de la clase delante, para ocultarlos y evitar su uso. Pero podr√≠amos llamarlos de la siguiente manera, aunque por lo general no es una buena idea:

```python
print(p._Persona__nombre) # Acceso al atributo privado con el nombre modificado
p._Persona__saludar() # Acceso al m√©todo privado con el nombre modificado

# Salida:
# Ana
# Hola, soy Ana y tengo 25 a√±os.
```
